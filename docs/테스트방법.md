● 백엔드를 테스트하는 방법은 두 가지가 있습니다:

  방법 1: Docker Compose 사용 (권장)

  전체 스택(DB + Backend + Frontend)을 한 번에 실행:

  # 프로젝트 루트에서
  cd /home/nuricom/project/myappStore

  # Docker Compose로 모든 서비스 시작
  docker-compose up -d

  # 로그 확인
  docker-compose logs -f backend

  # 중지
  docker-compose down

  - 백엔드 주소: http://localhost:8100
  - API 문서: http://localhost:8100/docs (Swagger UI)
  - 프론트엔드: http://localhost:5900

  방법 2: 로컬 직접 실행 (개발용)

  더 빠른 개발/테스트를 위한 방법:

  # 1. PostgreSQL 데이터베이스만 Docker로 실행
  cd /home/nuricom/project/myappStore
  docker-compose up -d db

  # 2. 백엔드 디렉토리로 이동
  cd backend

  # 3. 가상환경 활성화 (이미 venv가 있으면)
  source venv/bin/activate

  # 4. 의존성 설치 (처음 한 번만)
  pip install -r requirements.txt

  # 5. 백엔드 실행
  uvicorn app.main:app --host 0.0.0.0 --port 8100 --reload

  --host 0.0.0.0 옵션: 모든 네트워크 인터페이스에서 수신 (localhost 포함)
  --reload 옵션: 코드 변경 시 자동으로 재시작되어 개발에 편리합니다.

lsof -ti:8100 | xargs kill -9

sudo lsof -t -i:8100
netstat -nap|grep 8100

  데이터베이스 마이그레이션

  처음 실행 시 DB 테이블 생성이 필요할 수 있습니다:

  cd /home/nuricom/project/myappStore/backend

  # Alembic 마이그레이션 실행
  alembic upgrade head

  환경 변수 확인

  backend/.env 파일에 다음 설정이 있는지 확인:
  - DATABASE_URL: PostgreSQL 연결 정보
  - SECRET_KEY: JWT 암호화 키
  - OPENAI_API_KEY: (선택) AI 메타데이터 생성용

  API 테스트

  실행 후 브라우저에서:
  - Swagger UI: http://localhost:8100/docs
  - ReDoc: http://localhost:8100/redoc
  - Health Check: http://localhost:8100/api/products/stats/overview
